Name Match

Project Overview

One of the toughest and most common challenges in social science research is identifying records that refer to the same person. This is necessary when linking multiple datasets together (i.e. to determine which individuals in dataset A appear in dataset B) and when deduplicating a single dataset (i.e. to determine how many times a given person appears in dataset A). If any of the datasets involved are missing a unique person identifier (like social security number), typos, nicknames, name changes, and common names make this a surprisingly tricky problem.


To solve this, I helped develop Name Match, an open-source Python package that uses machine learning and user-defined constraints to efficiently match records across datasets with high accuracy. It compares fields like name, date of birth, address, and more, while giving users the flexibility to define what invalid matches looks like for their specific project.


At the Crime Lab, we've used Name Match to link records in several major predictive tasks and evaluations, including READI Chicago and One Summer Chicago. In 2023, we released Name Match on GitHub so other researchers and public sector teams can use it in their own work and directly contribute to its improvement. My Role

As an intern at the Crime Lab in Summer 2016, I was tasked with transforming an early prototype of probabilistic record linkage code—originally designed to link police records for the Domestic Violence in NYC project—into a flexible and performant Python tool for use in future Crime Lab projects. After joining the Crime Lab full-time in January 2017, I became the primary developer of the tool, now known as Name Match.


My responsibilities included designing and implementing new features, improving model accuracy, fixing bugs, and overseeing a data engineer's performance optimization efforts. I also used Name Match to perform record linkage for various Crime Lab projects, delivering linked datasets to research teams as needs arose. As internal demand grew, I focused on enhancing usability, writing documentation, tutorials, and user guides, and presenting training materials to colleagues. Eventually, I led efforts to open-source Name Match on GitHub, making it accessible to external researchers.


Separately, I've leveraged my expertise in record linkage to quantify the value of having labeled data when using off-the-shelf record linkage tools. Together with co-authors Zubin Jelveh, Sarah Tahamont, Shi Yan, Aaron Chalfin, and Benjamin Hansen, we published No Ground Truth? No Problem: Improving administrative data linking using active learning and a little bit of guile to show that researchers can achieve high-quality linkage results with relatively few labeled examples of matching and non-matching records.
